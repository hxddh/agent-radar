### 5. Playbook Candidates

- Trick: Treat unfamiliar repositories as hostile build inputs before allowing an agent to run setup commands.
  - When useful: Any coding agent task involving a newly cloned repo, package install, setup script, Makefile, postinstall hook, or network-enabled command.
  - Evidence: 0DIN/Mozilla proof-of-concept coverage shows a clean-looking repo can lead an agent down an error-recovery path that retrieves executable payloads at runtime; secondary security analysis argues the trust-boundary issue applies to any shell-capable coding agent.
  - Should promote to playbook? yes.
  - Source: https://www.tomshardware.com/tech-industry/cyber-security/ai-coding-agents-can-be-tricked-into-installing-malware-via-clean-github-repositories-mozillas-0din-team-shows-how-claude-code-can-be-exploited-by-its-own-helpfulness and https://hivesecurity.gitlab.io/blog/claude-code-clean-repo-trap/
