# Agent Radar Monthly - 2026-07

Partial-month review based on the first operating day of the radar, 2026-07-02. This is a seed review, not a full July conclusion.

## 1. Executive Summary

- Biggest thesis change: No thesis change yet; the first signals strengthen the existing view that agent infrastructure is becoming the core story.
- Strongest product signal: Browser tools, sandbox execution, terminal routing, remote agent threads, and enterprise workflow integrations are moving into mainstream product surfaces.
- Strongest user-experience signal: Developers may mix agents based on quota, work context, and task type, but current evidence is still weak and anecdotal.
- Strongest infrastructure signal: Agent runtimes now include browser sessions, MCP tools, Slack/Linear triggers, sandbox snapshots, temporary deployments, remote threads, memory, and logs.
- Strongest storage implication: Agent state is spreading across workspaces, session history, traces, screenshots, memory, enterprise knowledge, deployment artifacts, and feedback loops.
- Biggest anti-signal: Shell-capable coding agents remain risky when they can run setup commands in unfamiliar repositories without strong trust boundaries.

## 2. Watchlist Changes

- Added source-backed notes for Codex, Claude Code, Cursor, GitHub Copilot, Devin, Replit Agent, Warp, Amp, Factory, Raycast AI, Vercel Sandbox, and Cloudflare agent workflows.
- Kept weaker emerging candidates such as OpenCode and Pi in watch mode only.
- Remaining watchlist gaps: more real user evidence is needed for Devin, Replit Agent, Warp, Amp, Factory, Raycast AI, and several smaller coding-agent candidates.

## 3. Evidence Quality Review

- Strong evidence: Official changelogs and docs from OpenAI, GitHub, Cursor, Devin, Replit, Warp, Amp, Factory, Raycast, Vercel, and Cloudflare.
- Medium evidence: Secondary security analysis around the 0DIN clean-repo attack, supported by multiple public writeups.
- Weak evidence: Reddit comments about multi-tool usage and emerging tools. These are useful as field notes, not conclusions.

## 4. Playbook Promotions

- Promoted "Untrusted Repository Setup" into `playbook.md`.
- Rationale: The clean-repo attack pattern is high-severity, structurally relevant across shell-capable coding agents, and directly actionable.

## 5. Storage and Infrastructure Thesis

- Strengthened: Agent storage is not only long-term memory. It includes logs, traces, screenshots, browser state, sandbox files, snapshots, temporary deployments, session messages, enterprise knowledge, and replayable work history.
- Watch: Whether providers expose retention, redaction, export, and audit controls as first-class product features.

## 6. Commercialization and Enterprise Adoption

- Factory's "software factory" framing and Devin's Slack/Linear/GHES integrations are the clearest enterprise-positioning signals.
- GitHub Copilot browser controls and Raycast team/shared AI surfaces suggest governance is becoming a buying criterion.
- Commercial evidence remains incomplete without pricing, usage, renewal, and customer field data.

## 7. Security and Governance

- Strongest signal: Untrusted repository setup is now a core agent security concern.
- Governance surfaces to track: sandbox defaults, shell approval, network egress, MCP permissions, browser sharing, memory controls, log redaction, and enterprise audit trails.

## 8. Open Questions Resolved

- None yet. The radar is too new to close any major open question.

## 9. Open Questions Added

- Will terminal apps become general agent clients that coordinate many underlying CLI agents?
- Will desktop agents such as Raycast become trusted memory layers for everyday work?
- Will "software factory" positioning become a concrete operating model or remain enterprise narrative?
- Which providers will make log/traces redaction and replay governance first-class?

## 10. Next Month Watchlist

- More real user evidence for Devin, Replit, Warp, Amp, Factory, and Raycast.
- Agent security mitigations after clean-repo attack coverage.
- Storage retention controls for logs, traces, browser artifacts, and memories.
- Pricing and quota changes that affect agent usage behavior.
- Cross-agent handoff patterns such as Claude-to-Replit and terminal-to-CLI-agent routing.
