# Agent Radar Monthly - 2026-07
Partial-month review based on the first operating day of the radar, 2026-07-02. This is a seed review, not a full July conclusion.

> Format: read the full `## English` section first, then the full `## 中文` section. URLs, repo names, and product names appear once in the English section unless language-neutral.

## English

### 1. Executive Summary

- Biggest thesis change: No thesis change yet; the first signals strengthen the existing view that agent infrastructure is becoming the core story.

- Strongest product signal: Browser tools, sandbox execution, terminal routing, remote agent threads, and enterprise workflow integrations are moving into mainstream product surfaces.

- Strongest user-experience signal: Developers may mix agents based on quota, work context, and task type, but current evidence is still weak and anecdotal.

- Strongest infrastructure signal: Agent runtimes now include browser sessions, MCP tools, Slack/Linear triggers, sandbox snapshots, temporary deployments, remote threads, memory, and logs.

- Strongest storage implication: Agent state is spreading across workspaces, session history, traces, screenshots, memory, enterprise knowledge, deployment artifacts, and feedback loops.

- Biggest anti-signal: Shell-capable coding agents remain risky when they can run setup commands in unfamiliar repositories without strong trust boundaries.

### 2. Watchlist Changes

- Added entries: Added source-backed notes for Codex, Claude Code, Cursor, GitHub Copilot, Devin, Replit Agent, Warp, Amp, Factory, Raycast AI, Vercel Sandbox, and Cloudflare agent workflows.

- Watch-only candidates: Kept weaker emerging candidates such as OpenCode and Pi in watch mode only.

- Remaining gaps: Remaining watchlist gaps: more real user evidence is needed for Devin, Replit Agent, Warp, Amp, Factory, Raycast AI, and several smaller coding-agent candidates.

### 3. Evidence Quality Review

- Strong evidence: Official changelogs and docs from OpenAI, GitHub, Cursor, Devin, Replit, Warp, Amp, Factory, Raycast, Vercel, and Cloudflare.

- Medium evidence: Secondary security analysis around the 0DIN clean-repo attack, supported by multiple public writeups.

- Weak evidence: Reddit comments about multi-tool usage and emerging tools. These are useful as field notes, not conclusions.

### 4. Playbook Promotions

- Promotion: Promoted "Untrusted Repository Setup" into `playbook.md`.

- Rationale: The clean-repo attack pattern is high-severity, structurally relevant across shell-capable coding agents, and directly actionable.

### 5. Storage and Infrastructure Thesis

- Strengthened: Agent storage is not only long-term memory. It includes logs, traces, screenshots, browser state, sandbox files, snapshots, temporary deployments, session messages, enterprise knowledge, and replayable work history.

- Watch: Whether providers expose retention, redaction, export, and audit controls as first-class product features.

### 6. Commercialization and Enterprise Adoption

- Enterprise positioning: Factory's "software factory" framing and Devin's Slack/Linear/GHES integrations are the clearest enterprise-positioning signals.

- Governance as buying criterion: GitHub Copilot browser controls and Raycast team/shared AI surfaces suggest governance is becoming a buying criterion.

- Evidence gap: Commercial evidence remains incomplete without pricing, usage, renewal, and customer field data.

### 7. Security and Governance

- Strongest signal: Untrusted repository setup is now a core agent security concern.

- Governance surfaces to track: Governance surfaces to track: sandbox defaults, shell approval, network egress, MCP permissions, browser sharing, memory controls, log redaction, and enterprise audit trails.

### 8. Open Questions Resolved

- Status: None yet. The radar is too new to close any major open question.

### 9. Open Questions Added

- Terminal as agent client: Will terminal apps become general agent clients that coordinate many underlying CLI agents?

- Desktop memory layer: Will desktop agents such as Raycast become trusted memory layers for everyday work?

- Software factory reality: Will "software factory" positioning become a concrete operating model or remain enterprise narrative?

- Log governance: Which providers will make log/traces redaction and replay governance first-class?

### 10. Next Month Watchlist

- User evidence: More real user evidence for Devin, Replit, Warp, Amp, Factory, and Raycast.

- Security mitigations: Agent security mitigations after clean-repo attack coverage.

- Storage retention controls: Storage retention controls for logs, traces, browser artifacts, and memories.

- Pricing and quota: Pricing and quota changes that affect agent usage behavior.

- Cross-agent handoff: Cross-agent handoff patterns such as Claude-to-Replit and terminal-to-CLI-agent routing.

## 中文

### 1. Executive Summary

- 尚无 thesis 变化；首批信号强化了“Agent 基础设施正在成为核心叙事”的既有判断。

- 浏览器工具、沙箱执行、终端路由、远程 Agent 线程与企业工作流集成，正在进入主流产品界面。

- 开发者可能按配额、工作场景和任务类型混用 Agent，但证据仍弱且偏轶事。

- Agent 运行时已覆盖浏览器会话、MCP 工具、Slack/Linear 触发器、沙箱快照、临时部署、远程线程、记忆与日志。

- Agent 状态正扩散到工作区、会话历史、trace、截图、记忆、企业知识、部署产物与反馈回路。

- 具备 shell 能力的编程 Agent 在陌生仓库中执行 setup 命令时，若缺少强信任边界，风险仍然很高。

### 2. Watchlist Changes

- 为 Codex、Claude Code、Cursor、GitHub Copilot、Devin、Replit Agent、Warp、Amp、Factory、Raycast AI、Vercel Sandbox 和 Cloudflare Agent 工作流补充了有来源支撑的条目。

- OpenCode、Pi 等较弱的新兴候选仅保留在观察模式。

- Devin、Replit Agent、Warp、Amp、Factory、Raycast AI 以及若干较小的编程 Agent 候选仍需要更多真实用户证据。

### 3. Evidence Quality Review

- 来自 OpenAI、GitHub、Cursor、Devin、Replit、Warp、Amp、Factory、Raycast、Vercel 和 Cloudflare 的官方更新日志与文档。

- 围绕 0DIN 干净仓库攻击的二手安全分析，有多篇公开文章佐证。

- 关于多工具混用与新兴工具的 Reddit 评论。它们可作为田野笔记，但不能作为结论。

### 4. Playbook Promotions

- 将“不受信任仓库的初始化”（Untrusted Repository Setup）提升进 `playbook.md`。

- 干净仓库攻击模式严重性高，对所有具备 shell 能力的编程 Agent 都有结构性影响，且可直接付诸行动。

### 5. Storage and Infrastructure Thesis

- Agent 存储不只是长期记忆，还包括日志、trace、截图、浏览器状态、沙箱文件、快照、临时部署、会话消息、企业知识与可回放的工作历史。

- 观察各提供商是否会把保留、脱敏、导出与审计控制作为一级产品能力暴露出来。

### 6. Commercialization and Enterprise Adoption

- Factory 的“软件工厂”叙事与 Devin 的 Slack/Linear/GHES 集成是最清晰的企业定位信号。

- GitHub Copilot 浏览器控制与 Raycast 团队/共享 AI 界面表明，治理正在成为采购标准之一。

- 在缺少定价、用量、续费与客户田野数据之前，商业化证据仍不完整。

### 7. Security and Governance

- 不受信任仓库的初始化如今已是核心的 Agent 安全议题。

- 需要跟踪的治理界面包括沙箱默认值、shell 审批、网络出口、MCP 权限、浏览器共享、记忆控制、日志脱敏与企业审计日志。

### 8. Open Questions Resolved

- 尚无。雷达刚启动，还不足以关闭任何重要的开放问题。

### 9. Open Questions Added

- 终端应用会不会成为协调多个底层 CLI Agent 的通用 Agent 客户端？

- Raycast 这类桌面 Agent 会不会成为日常工作可信赖的记忆层？

- “软件工厂”定位会成为具体的运营模式，还是停留在企业叙事层面？

- 哪些提供商会把日志/trace 脱敏与回放治理做成一级能力？

### 10. Next Month Watchlist

- 为 Devin、Replit、Warp、Amp、Factory 和 Raycast 收集更多真实用户证据。

- 干净仓库攻击曝光后各 Agent 的安全缓解措施。

- 日志、trace、浏览器产物与记忆的存储保留控制。

- 影响 Agent 使用行为的定价与配额变化。

- 跨 Agent 交接模式，例如 Claude 到 Replit、终端到 CLI Agent 的路由。
