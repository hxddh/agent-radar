## 6. Useful Tricks

- 中文：
- English: Promote "Untrusted Repository Setup" to the playbook.
- 中文：
- English: Core pattern: before allowing an agent to run setup commands in an unfamiliar repo, inspect bootstrap paths, package scripts, Makefiles, network calls, and postinstall hooks.
- 中文：
- English: Practical default: use disposable sandboxes with restricted secrets and limited network egress for unknown codebases.

## 8. Storage Implications

- 中文：
- English: Logs and traces are sensitive. Codex's trace-log fix is a concrete reminder that agent observability may store prompts, code, credentials, or private context.
- 中文：
- English: Browser artifacts are storage artifacts. Screenshots, console output, and page state may need retention, redaction, and access policies.
- 中文：
- English: Sandboxes are workspace products. Persistence, snapshots, drives, live previews, and file edits are storage primitives, not incidental implementation details.
- 中文：
- English: Temporary deploys create artifact lifecycle questions: when to expire, when to claim, how to audit, and how to move from ephemeral to persistent state.
- 中文：
- English: Enterprise agent workflows create additional storage surfaces: Slack thread context, Linear triggers, MCP logs, Axiom observability data, session messages, remote agent threads, AI memory, profile data, and software-factory feedback loops.
- 中文：
- English: Agent memory as a distinct storage workload: Multiple open-source projects target persistent, semantic, and local-first memory for agents, reinforcing the thesis that memory is a core agent layer.
