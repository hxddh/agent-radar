Task:
Update the Agent Watchlist.

Use public sources, authorized logged-in sources, and user-provided private signals when available. Keep public output source-classified and public-safe.

For mainstream agents, evaluate:
1. Repo understanding
2. Long-task ability
3. Tool calling
4. Task closure
5. User control
6. Reliability
7. Enterprise readiness
8. Cost experience
9. User sentiment

For emerging agents, evaluate:
1. New entry point
2. New scenario
3. New runtime / sandbox / memory architecture
4. New interaction model
5. New distribution path
6. New user group
7. Chance of being absorbed by mainstream agents

Rules:
- Do not add every new project.
- Add only if it signals a meaningful new direction.
- Keep each agent concise.
- Preserve old notes unless contradicted by new evidence.
- Include source links, source classes, or source status labels.
- Label weak evidence as weak instead of blocking.
- Anonymize private or logged-in evidence before publishing.
